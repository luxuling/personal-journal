This site. The one you're reading right now.

## Why

I wanted something simple: a place where people can actually stay and read without friction, and
where I can keep updating my programming journey as it unfolds.

## Stack

- **Astro** — page structure and static output
- **SolidJS** — interactive islands (nav, toggles)
- **Tailwind v4** — utility styles
- **TypeScript** — types everywhere

## What's interesting

I went with **Astro** instead of **Next.js**—I was tired of the Next.js shape of things for a
site that's mostly pages and prose. For me, MDX (including remote MDX) is easier to reason about
and pipe through Astro than it was in the setups I kept reaching for with Next. **SolidJS** only
shows up where something actually needs to run in the browser; that island model keeps client-side
work smaller and simpler than treating the whole site like an app.

Astro's content collections still back the structured data (this page, for example), and MDX means
I can write richer descriptions without fighting JSON or hardcoded HTML.

The GitHub contribution heatmap pulls live data at build time via the GitHub GraphQL API —
no client-side fetch, no loading state.
